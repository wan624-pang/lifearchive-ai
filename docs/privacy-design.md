# LifeArchive AI — 隐私设计文档

## 设计原则

LifeArchive AI 处理的都是用户最私密的个人文件：身份证、合同、病历、保险、财务记录。隐私不是附加功能，而是核心设计约束。

## 1. 数据最小化

**原则**：只处理用户主动上传的文件。

- 系统不会自动扫描设备上的文件
- 不会访问用户的网盘、邮箱或云端存储
- 所有处理都基于用户明确的上传操作
- 不收集设备信息、浏览历史或其他无关数据

## 2. 本地优先

**原则**：数据存储和处理优先在本地完成。

- MVP 使用本地 SQLite 数据库
- 文件解析（PDF、DOCX、TXT）在本地完成
- Mock 模式下的分类完全在本地运行
- 后续版本将支持本地向量数据库（如 ChromaDB）
- 即使使用 LLM API，原始文件也存储在本地

## 3. 敏感信息保护

**原则**：自动识别并标记敏感信息。

系统会标记以下类型的敏感信息：
- 身份证号
- 手机号码
- 银行卡号
- 家庭住址
- 医疗信息
- 密码和账户信息

敏感等级分类：
- **high**：身份与证件、合同与法律、医疗与健康、保险与资产
- **medium**：房屋与车辆、学习与证书、工作资料、发票与报销
- **low**：家庭纪念、旅行与照片

后续版本将支持：
- 敏感字段脱敏展示（如身份证号显示为 310***1234）
- 敏感文件单独加密存储

## 4. AI 可解释

**原则**：AI 结果透明，用户始终有控制权。

- 所有分类结果都显示置信度（confidence: 0.0-1.0）
- 低置信度结果标记为需要人工确认（needs_review: true）
- 用户可以查看分类依据
- AI 只是辅助工具，最终决定权在用户
- 分类错误不会导致数据丢失

## 5. 不保存密码

**原则**：识别到敏感账号资料时，不展示密码明文。

- 如果文件包含密码、账户信息等内容
- 系统只标记为「敏感账号资料」
- 不在摘要、标签或任何界面中展示密码原文
- 不在搜索结果中暴露密码信息

## 6. 可删除

**原则**：用户对自己的数据有完全控制权。

- 可以删除整个资料库
- 可以删除单个文件
- 删除操作不可逆，数据彻底清除
- 后续版本支持一键清除所有数据

## 7. 边界声明

**原则**：明确产品边界，不提供专业建议。

- 本产品不提供法律建议
- 本产品不提供医疗建议
- 本产品不提供财务建议
- 只做资料整理和检索辅助
- 涉及重要决策时，请咨询专业人士

## 8. LLM 数据处理

当使用真实 LLM API 时：

- 只发送文件文本内容用于分类，不发送文件本身
- 不使用用户数据进行模型训练
- API 调用遵循供应商的数据处理协议
- 用户可以选择仅使用 Mock 模式，完全离线

## 9. 后续安全计划

- [ ] 端侧模型：使用本地 LLM 完成所有 AI 处理
- [ ] 文件加密：对敏感文件进行加密存储
- [ ] 访问控制：支持密码保护和多用户隔离
- [ ] 审计日志：记录所有数据访问和修改操作
- [ ] 数据导出：支持导出所有数据并清除服务器数据
