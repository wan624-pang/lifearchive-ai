# Token 计划申请材料

## 项目名称

LifeArchive AI：个人与家庭数字资料库整理助手

## 项目简介

LifeArchive AI 是一个帮助用户整理个人和家庭数字资料的 AI 工具。用户上传一个包含各种文件的压缩包，系统自动完成文件解析、分类、摘要、标签提取、关键信息识别和资料库报告生成，把混乱的个人资料变成一个可搜索、可交接的私人档案馆。

## 解决的问题

普通人的数字生活中积累了大量重要文件——合同、病历、保险保单、毕业证书、发票、房产证、体检报告……这些文件散落在手机相册、微信文件传输助手、电脑下载文件夹、邮箱附件、网盘各个角落。

问题不只是「找不到」，更是「理不清」：
- 文件名是「IMG_20240315.jpg」或「新建文档(3).docx」
- 不知道保险什么时候到期，合同还剩几个月
- 搬家时不知道哪些资料需要带走
- 帮父母整理资料时无从下手
- 出国需要公证材料时到处翻找

传统方法是手动建文件夹分类，但文件太多、类型复杂、命名混乱，大多数人坚持不下来。

## 为什么需要 AI

这个场景天然需要 AI，原因有三：

1. **文件类型复杂**：同一个资料库里可能有 PDF 合同、图片收据、Word 文档、纯文本笔记，需要理解不同格式的内容
2. **命名不规范**：用户不会把文件命名为「2024年上海市浦东新区租房合同」，而是「合同.pdf」或「扫描件001.jpg」
3. **需要语义理解**：区分一份文件是「保险保单」还是「保险推销传单」，靠关键词不够，需要理解文件内容

关键词搜索和文件夹管理不够用。你需要一个能「读懂」文件内容的系统。

## 为什么需要大量 Token

LifeArchive AI 的 token 消耗主要来自：

1. **长文档处理**：合同、体检报告、保险保单等文件通常有几百到几千字，每个文件都需要完整的上下文输入
2. **多轮处理流程**：每个文件要经过分类、摘要、标签提取、关键信息识别等多步处理
3. **跨文件分析**：生成资料库报告时需要综合所有文件的分类结果，分析缺失材料、重复文件、重要时间线
4. **自然语言搜索**：每次搜索都需要解析用户意图，理解「找出所有保险相关文件」这类查询
5. **持续迭代**：优化 prompt、测试不同类型的文档、评估分类准确率，这些都需要大量试验

以一个包含 50 个文件的典型资料库为例，仅分类就需要 50 次 LLM 调用，加上报告生成和搜索，一次完整的处理流程需要约 200,000-500,000 tokens。

## 当前 MVP 已完成

- ZIP 上传和多格式文件解析（txt, md, pdf, docx, 图片）
- Mock AI 分类器（可在无 API key 时运行 demo）
- OpenAI-compatible LLM 客户端（可接入任何兼容 API）
- 11 种文件类别的自动分类
- 文件摘要和关键词标签生成
- 涉及人物和机构识别
- 重要日期提取（支持多种日期格式）
- 敏感等级判断
- 资料库总览报告（含统计、时间线、缺失提醒、交接清单）
- 自然语言搜索
- Web Dashboard（首页、资料库、文件详情、报告、隐私说明）
- 8 个虚构示例文件
- 完整的 README 和文档

## 项目价值

这不是一次性 demo。LifeArchive AI 解决的是一个真实的、普遍的问题：每个人的数字生活中都有大量重要文件需要整理和管理。

适用场景：
- 个人资料整理和备份
- 家庭资产管理
- 留学生出国材料准备
- 自由职业者财务资料管理
- 老人照护和家庭资料交接
- 搬家、换工作时的资料迁移

项目代码开源，可以在 GitHub 上持续维护和发展。

## Token 使用计划

获得的 token 将用于：

1. **文档分类测试**：用真实 LLM 替换 Mock 分类器，测试不同文档类型的分类准确率
2. **长上下文处理**：处理真实的合同、报告等长文档，验证大上下文场景下的分类质量
3. **多轮摘要优化**：迭代 prompt 设计，提升摘要质量
4. **资料库报告生成**：测试综合多文件的报告生成能力
5. **搜索意图解析**：优化自然语言搜索的意图理解
6. **多模态测试**：测试图片 OCR 后的文字分析能力
7. **评测和基准测试**：构建评测集，量化分类准确率和摘要质量
